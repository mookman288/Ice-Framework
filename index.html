Purpose Statement

The Ice Framework is a modifiable and reusable proof of concept designed to provide a basic structure for implementing fluid and accessible website designs. Originally, Ice was designed to be a working alternative to other modern frameworks that focused too heavily on pixel-based grid systems. However, as Ice has progressed over time, it has become abundantly clear that frameworks should not try and reinvent design, but to provide a fast and scalable groundwork for the imagination to take advantage of.

Installation

Ice Framework comes packaged compiled and compressed for implementation. The following table describes what files are required for implementation, where they should be located, and what they are designed to accomplish. This example page can act as an HTML template outlining the structure Ice expects, as well as a how-to on implementing the Ice Framework.

File Description Required
css/stylesheet.css Pre-compiled stylesheet containing all rules. Yes
js/modernizr.js Pre-compiled custom Modernizr build. No
js/vendor.js Compilation of vendor libraries. No
js/framework.js Pre-compiled framework containing all JS. No
fonts/* All of the necessary fonts for basic use. No

Here's an example of how a basic page structure might look: 

<!DOCTYPE html>
<html>
	<head>
		<title>Title Tag</title>
		<meta charset="utf-8" />
		<meta http-equiv="X-UA-Compatible" content="IE=edge, chrome=1" />
		<meta name="HandheldFriendly" content="true" />
		<meta name="MobileOptimized" content="320" />
		<meta name="viewport" content="width=device-width, initial-scale=1.0" />
		<link rel="icon" href="images/favicon.png" />
		<link rel="shortcut icon" href="favicon.ico" />
		<link rel="stylesheet" type="text/css" href="css/stylesheet.css" />
	</head>
	<body>
		<-- Replace this with your custom HTML. -->
		<script type="text/javascript" src="js/modernizr.js"></script>
		<script type="text/javascript" src="js/vendor.js"></script>
		<script type="text/javascript" src="js/framework.js"></script>
	</body>
</html>

Modifications

Because Ice is built to be a framework, and does not come packaged with a design, it's up to the developer how they want to modify or extend the framework. The core of the framework is entirely based on Sass, and is run through the task runner known as Gulp. Gulp is written to work with Node.js. If you're not familiar with these technologies, take a look at Travis Maynard's article here.

If you have npm or yarn previously installed, once you've cloned this repository, you should simply need to install the dependencies found in the package.json file. Then you can begin working by calling gulp for production, or gulp --dev for development.

Most common grid systems are designed around an arbitrary standard of 960px maximum width. This, coupled with the adoption of a mobile-first design strategy, has once again put pixel perfect design above organic architectural design. This methodology has inadvertently forced websites to cut back on innovative functionality in favor of simplistic form. Although effective on smaller resolution devices, it is the opinion of this author that this has significantly deteriorated the browsing experience on higher resolution devices. Ultimately, what these grid systems lack is functional scalability across a plethora of device widths.

Despite the fact that developing an accessible experience across a vast number of device widths is a complicated and difficult task, abandoning major functionality to attain said accessibility is not necessary. Ice hopes to introduce a conceptual argument in favor of highly scalable grid systems that are geared toward a desktop first experience. In doing so, it is expected that developers will be able to maintain a rich browsing experience for higher resolution devices, while also catering to the simplistic expectations found in smaller resolution devices.

In order to accomplish this level of scalability, five major breakpoints have been selected based on a variety of device width data as of Q1 2016. In order of greatest to least:

Description Selector Breakpoint Font Size Multiplier Example
High Resolution hd 2160 1.60 hd-6
Desktop Resolution des 1440 1.60 des-8
Standard Devices std 1000 1.40 std-10
Tablet Devices tab 768 1.40 tab-12
Mobile Devices mob 640 1.20 mob-14
Legacy Phones tel 384 1.20 tel-16
